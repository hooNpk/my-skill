# Google Sheets Reservation System

Google Sheets를 백엔드로 사용하는 예약 시스템을 구축하는 스킬입니다. 예약 신청 폼, 예약 내역 게시판, 스케줄 캘린더 3가지 컴포넌트로 구성됩니다.

## 이 스킬이 하는 일

별도 서버 없이 Google Sheets와 Apps Script만으로 완전한 예약 시스템을 만듭니다:

1. **예약 신청 폼** — 사용자가 예약 정보를 입력하고 제출
2. **예약 내역 게시판** — 제출된 예약 목록을 확인
3. **스케줄 캘린더** — 확정된 예약을 월간 캘린더로 표시

## 시스템 구조

```
┌──────────────┐     POST      ┌──────────────────┐    insertRow    ┌───────────────┐
│ 웹사이트 폼   │ ──────────→  │ Apps Script 웹앱  │ ────────────→  │ Google Sheets │
└──────────────┘               └──────────────────┘                 └───────┬───────┘
                                                                           │
┌──────────────┐   Sheets API  ┌──────────────────┐                        │
│ 웹사이트 게시판│ ←──────────  │ Google Sheets API │ ←──────── read ────────┘
│ 웹사이트 캘린더│              └──────────────────┘
└──────────────┘
```

### 왜 Google Forms를 안 쓰나요?

Google Forms의 `/formResponse` 엔드포인트는 외부 POST 제출 시 400 에러를 자주 반환합니다. 드롭다운 필드의 값 검증, `fbzx` 세션 토큰 요구 등으로 인해 불안정합니다. Apps Script 웹앱은 SpreadsheetApp API를 직접 사용하므로 이런 문제가 없습니다.

## 사전 준비

### 1. Google Sheets 생성

예약용 스프레드시트를 만들고, 1행에 헤더를 설정합니다:

```
타임스탬프 | 이름 | 연락처 | 희망 날짜 | 시간대 | 룸선택 | 인원 | 촬영목적 | 입금자명 | 방문차량 | 장비렌탈 | 특이사항 | 비밀번호 | 공개여부 | 상태
```

스프레드시트를 **"링크가 있는 모든 사용자 — 뷰어"** 로 공유합니다 (게시판에서 읽기 위해 필요).

### 2. Google Sheets API 키 발급

1. [Google Cloud Console](https://console.cloud.google.com) 접속
2. API 및 서비스 → 라이브러리 → "Google Sheets API" 활성화
3. 사용자 인증 정보 → API 키 생성
4. 키 제한: Google Sheets API만 허용 권장

### 3. Apps Script 웹앱 배포

1. 스프레드시트 열기 → 확장 프로그램 → Apps Script
2. `references/apps-script.md`의 코드를 붙여넣기
3. `SHEET_ID`를 실제 스프레드시트 ID로 변경
4. 배포 → 새 배포 → 유형: **웹 앱**
5. 실행 계정: **나** / 액세스 권한: **모든 사용자**
6. 배포 후 웹앱 URL 복사

### 4. config.js 설정

```javascript
const CONFIG = {
  GOOGLE_SHEETS_API_KEY: "AIzaSy...",
  RESERVATION_SHEET_ID: "스프레드시트ID",
  APPS_SCRIPT_URL: "https://script.google.com/macros/s/.../exec",
};
```

## 컴포넌트별 상세

### 1. 예약 신청 폼

사용자가 예약 정보를 입력하는 HTML 폼입니다.

**주요 기능:**

| 기능 | 설명 |
|------|------|
| 시간 선택 | 시작/종료 시간 2개 드롭다운 (09:00~20:00, 30분 간격) |
| 시간 검증 | 종료 시간이 시작 시간보다 늦은지 자동 검증 |
| CAPTCHA | 간단한 산술 문제 (예: "3 + 7 = ?")로 자동등록 방지 |
| 개인정보 동의 | 체크박스로 수집/이용 동의 필수 확인 |
| 비밀글 | 토글 ON 시 비밀번호 입력, 게시판에서 비밀번호 확인 후 열람 |
| 비동기 제출 | fire-and-forget 방식 — 응답을 기다리지 않아 즉시 완료 |

**fire-and-forget 제출 방식:**

```javascript
// Apps Script 응답을 기다리지 않음 (3~5초 절약)
fetch(CONFIG.APPS_SCRIPT_URL, {
  method: 'POST',
  mode: 'no-cors',
  headers: { 'Content-Type': 'application/x-www-form-urlencoded' },
  body: params.toString(),
});
// 즉시 완료 처리
alert('예약 신청이 완료되었습니다.');
allData.unshift({ /* 새 항목 */ });
renderList();
```

사용자 체감 속도가 0ms에 가까워집니다. 데이터는 백그라운드에서 스프레드시트에 기록됩니다.

### 2. 예약 내역 게시판

스프레드시트의 데이터를 읽어와 테이블로 표시합니다.

**주요 기능:**

| 기능 | 설명 |
|------|------|
| 페이지네이션 | 20개씩 표시, 이전/번호/다음 네비게이션 |
| 이름 마스킹 | `박정훈` → `박*훈`, `메가코드` → `메**드` |
| 상세 모달 | 행 클릭 시 전체 예약 정보 표시 |
| 비밀글 | 비밀번호 입력 후 열람 가능 |
| 최신순 정렬 | insertRow(2)로 저장되어 시트 순서 그대로 최신순 |
| 상태 표시 | '확인중' (빨강) / '예약완료' (초록) 뱃지 |

**데이터 읽기 — 탭 이름 자동 탐색:**

Google이 폼 연결 시 생성하는 탭 이름이 로캘에 따라 다릅니다. 여러 후보를 순회합니다:

```javascript
const tabCandidates = [
  '설문지 응답 시트1', '설문지 응답 시트2',
  '폼 응답 1', 'Form Responses 1', '시트1'
];
```

**이름 마스킹 구현:**

```javascript
function maskName(name) {
  if (name.length <= 2) return name[0] + '*';
  return name[0] + '*'.repeat(name.length - 2) + name[name.length - 1];
}
```

### 3. 스케줄 캘린더

예약 스프레드시트에서 **'예약완료' 상태만** 필터링하여 월간 캘린더에 표시합니다.

**주요 기능:**

| 기능 | 설명 |
|------|------|
| 데스크탑 뷰 | 월간 캘린더 그리드 (일~토) |
| 모바일 뷰 | 날짜별 세로 리스트 (예약 없는 날도 표시) |
| 룸별 색상 | A: 핑크, B: 파랑, C: 초록, A+B: 보라, 전체: 금색 |
| 예약자명 마스킹 | `박정훈` → `박**` |
| 월 이동 | ◀/▶ 버튼으로 이전/다음 월 이동 |

**데이터 필터링:**

```javascript
scheduleData = allData
  .filter(r => r['상태'] === '예약완료')
  .map(r => ({
    '날짜': r['희망 날짜'],
    '시간상세': r['시간대'],
    '룸': parseRoom(r['룸선택']),
    '예약자': maskName(r['이름']),
  }));
```

## 관리자 운영 방법

### 예약 확정하기

1. Google Sheets에서 예약 스프레드시트 열기
2. 해당 예약의 **상태** 열 (O열) 클릭
3. 드롭다운에서 **'예약완료'** 선택
4. 글자 색이 빨간색 → 초록색으로 자동 변경
5. 웹사이트 스케줄 캘린더에 자동 반영

### Q&A 답변하기

1. Q&A 스프레드시트 열기
2. 해당 질문의 **관리자답변** 열에 답변 작성
3. **답변일시** 열에 날짜 입력 (예: `2026-03-22`)
4. 웹사이트 Q&A 게시판에 답변이 자동 표시

### Apps Script 코드 수정 후 재배포

코드를 수정한 후에는 반드시 새 버전으로 배포해야 합니다:

1. Apps Script 에디터에서 코드 수정
2. 배포 → 배포 관리 → 연필(편집) 아이콘 클릭
3. 버전: **새 버전** 선택
4. 배포 클릭

URL은 동일하게 유지되므로 config.js를 수정할 필요가 없습니다.
